# Build Dashboard

Runs the **build-visualization** skill to assemble several charts into one coherent dashboard.

## Instructions
1. Take the metric or topic in `$ARGUMENTS` and list the 3–5 questions a viewer needs answered.
2. For each, recommend the right chart type from the data's shape and define clean encoding.
3. Render the panels in `~~notebook` (or output chart specs), keeping encodings consistent across them.
4. Arrange the dashboard most-important first, with takeaway titles on each panel.
5. Follow the **build-visualization** skill for chart-type mapping, house style, and layout.
